# blender-skills

Agent skills and a local MCP server for Blender headless automation, image
design, 3D file creation and instance-mask image edits.

Each skill is a separate directory so the agent loads only what it needs, not everything at once.

---

## Requirements

### Blender

| Requirement | Details |
|-------------|---------|
| **Blender version** | Skills retain broad compatibility; the MCP server targets **Blender 4.5 LTS+** |
| **Installation** | [blender.org/download](https://www.blender.org/download/) or via system package manager |
| **`blender` on PATH** | Run `blender --version` to verify; add to PATH if needed |

```bash
# Verify Blender is accessible
blender --version
```

### Python (inside Blender)

Blender ships its own Python interpreter — no separate Python install needed. Scripts run inside Blender's Python environment automatically.

All standard-library modules (`os`, `sys`, `math`, `glob`) are available. Third-party pip packages are **not** available by default unless installed into Blender's Python.

### Add-ons (optional, for specific skills)

| Add-on | Required by | Install |
|--------|-------------|---------|
| **Extra Objects** (`add_mesh_extra_objects`) | gear primitives and other mesh helpers | Blender add-ons bundle; enable via Preferences → Add-ons, `--addons add_mesh_extra_objects`, or set `BLENDER_MCP_BLENDER_ADDONS=add_mesh_extra_objects` |
| **3MF exporter** (`io_mesh_3mf`) | `blender-3d-export` (3MF format) | Bundled in Blender 3.x; enable via Preferences → Add-ons, or `--addons io_mesh_3mf` |
| **AMF exporter** | `blender-3d-export` (AMF format) | Third-party add-on — install manually, then `--addons <module_name>` |
| **Cycles** | `blender-rendering` (photo-realistic renders) | Bundled in Blender; enable via Preferences → Add-ons |

Skills that require add-ons include `try/except` fallbacks — missing add-ons print a warning but don't crash the script.

## Install skills

### Install all skills (recommended)

```bash
npx skills add <your-github-username>/blender-skills
```

This installs all skills into `~/.agents/skills/` where agent runtimes (Claude Code, etc.) can discover them.

### Install a single skill

```bash
npx skills add <your-github-username>/blender-skills/skills/blender-rendering
```

### Manual install

Copy any skill directory into `~/.agents/skills/`:

```bash
cp -r skills/blender-rendering ~/.agents/skills/
```

### Verify install

```bash
ls ~/.agents/skills/ | grep blender
```

## Quick start

For Codex, Claude Code or GitHub Copilot, prefer the versioned MCP server in
[`blender-mcp-server/`](./blender-mcp-server/README.md). Setup and client
configuration examples are in [`docs/mcp-server-setup.md`](./docs/mcp-server-setup.md).

Scripts generated by these skills run with:

```bash
blender --background --factory-startup --python your_script.py
```

With add-ons:
```bash
blender --background --factory-startup --addons io_mesh_3mf --python your_script.py
```

With custom arguments (available via `sys.argv` after `--`):
```bash
blender --background --factory-startup --python your_script.py -- /output/dir 0.5
```

## MCP Server

`blender-mcp-server` exposes typed `stdio` tools for versioned scenes,
materials, import/export, renders, Cryptomatte masks and optional YOLO11-seg
image edits. Scene operations call Blender in a child process and commit a new
`.blend` version only after success. External image mask edits run in Python
with Pillow and do not start Blender.

Use Cryptomatte for selecting known objects or materials in Blender renders.
Use YOLO segmentation for arbitrary external photographs only; it does not
establish an object identity in a 3D scene.

The optional Ultralytics integration in `blender-mcp-server` is distributed as
`AGPL-3.0-or-later`; the skill documents remain MIT licensed.

## Usage notes

- All skills target `blender --background` headless execution
- Version-sensitive properties always use `hasattr()` guards or `try/except`
- Exporter calls use probe pattern: current operator first, legacy fallback second
- Source: [Blender Python API](https://docs.blender.org/api/current/) + deep research on headless automation patterns

## Typical workflows

**3D print pipeline:** `blender-headless` → `blender-mesh-modeling` → `blender-modifiers` → `blender-mesh-repair` → `blender-3d-export`

**Render pipeline:** `blender-headless` → `blender-rendering` → `blender-compositing`

**Batch image processing:** `blender-headless` → `blender-image-editing`

**MCP external-image edit:** `image_segment` → select `instance_id` → `image_edit_by_mask`

**MCP exact scene mask:** `job_inspect` → `render_object_mask` (Cryptomatte)
